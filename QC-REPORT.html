1. Cropping Correction Report (the core fix)

Root cause: the image pipeline cover-cropped every room photo to a hard 3:2 with position:"attention" — that algorithm zooms into the bed and cuts headboards, wardrobes and room width. Several covers and the hero were portrait source photos (e.g. Super Deluxe IMG_0655 = 900×1200) force-cropped to landscape, showing only a sliver of the room.

Corrections applied:

  • No more cropping — rooms now resize fit:inside (full room, native aspect, max 1600×1200). Beds, headboards, tables, wardrobes, windows & layout all preserved.
  • Room detail galleries use full-image framing (object-contain) on clean white mats — nothing is ever cut; portrait + landscape sit together consistently.
  • Manifest reads real per-image dimensions so layout space is reserved correctly (no shift / CLS).
